# 26.1 Migration Reduction Notes

Date: 2026-06-13

This note records the current reduction/removal state while porting NeoECOAE to the Minecraft/NeoForge 26.1 line. It is a worktree snapshot, not a final release checklist.

## Keep These Decisions

- Core datagen stays. `NEDataGen` still registers server data through `NERegistryProvider`, and client/lang data through `NELangMergerProvider`.
- Core recipe generation stays for `EcoMachineRecipes` and `CoolingRecipes`.
- Optional external machine recipes are no longer generated from Java. The hand-maintained copies live under `src/main/resources/data/neoecoae/recipe`.
- Recipe builders should use provider lookups instead of constructing holders manually:
  - items: `provider.itemLookup()`
  - fluids: `provider.registries().lookupOrThrow(Registries.FLUID)`
- Integrated Working Station results use `ItemStackTemplate` / `FluidStackTemplate`. Do not port this back to runtime `ItemStack` / `FluidStack` codec state.
- Do not revive `Holder.direct(...)` or component binding workarounds for recipe results.
- Delombok / sourcesJar output is currently not part of this migration gate.

## Java Source Removed

These files are currently staged as deletions:

- `src/main/java/cn/dancingsnow/neoecoae/data/recipe/AAERecipes.java`
- `src/main/java/cn/dancingsnow/neoecoae/data/recipe/EAERecipes.java`
- `src/main/java/cn/dancingsnow/neoecoae/data/recipe/MekanismRecipes.java`

Reason: optional integration recipe datagen was removed from `NERecipeGenerator.accept(...)`. These recipes are either moved to manual resource JSON or should be restored later only after the integration APIs are ported.

- `src/main/java/cn/dancingsnow/neoecoae/data/model/CellModelGenerator.java`
- `src/main/java/cn/dancingsnow/neoecoae/registration/provider/NECellModelProvider.java`
- `src/main/java/cn/dancingsnow/neoecoae/registration/provider/NEProviderTypes.java`

Reason: old custom cell item model provider path was removed during the 26.1 model/datagen migration. Generated item model output now uses the new `assets/neoecoae/items` style output.

## Compile-Time Exclusions

`build.gradle` currently excludes several unported areas from `sourceSets.main.java`:

- client model/rendering helpers: `api/rendering/**`, `api/ECOCellModels.java`, `client/model/**`, `client/renderer/blockentity/**`, `client/all/NEBuiltinModels.java`, `client/all/NEExtraModels.java`
- model bakery / AAE mixins: `mixins/aae/**`, `mixins/ModelBakeryMixin.java`
- external integrations: `appbot`, `appflux`, `appmek`, `arsenergistique`, `emi`, `jade`, `jei`, `kubejs`, `xei`

Treat these as temporary migration cuts unless a later decision says the feature is permanently dropped.

Current groups:

- `crushing`: 6 recipes

Current generated-resource deletions include:

- old item models under `src/generated/resources/assets/neoecoae/models/item`: 78 files
- old block models under `src/generated/resources/assets/neoecoae/models/block`: 7 files
- generated unfinished lang scratch files under `assets/neoecoae/lang/unfinished`: 4 files

Generated recipe JSON deletions are split into two groups:

- staged manual-resource move: `circuit_cutter`, `crushing`, `crystal_assembler`, `crystal_fixer`, `enriching`, `reaction_chamber`
- unstaged runData side effects: root `eco_chemical` / `eco_fe` / `eco_mana` / `eco_source` recipes, `cooling`, `disassembly`, `enchanting_apparatus`, and `mana_infusion`

Before finalizing a commit, review the unstaged side effects separately. They are not the same decision as the manually moved optional recipes.

## Porting Notes

- When restoring optional integrations, prefer resource JSON with conditions unless a Java generator is still clearly useful.
- If Java generation is restored, pass `HolderGetter<Item>` / `HolderGetter<Fluid>` from the provider into builders.
- Keep `src/main/resources/data` owned by hand-maintained optional recipes. Do not let `runData` overwrite it.
- Re-check excluded integration/client packages one by one after their upstream APIs are available on 26.1.
- Re-run `gradlew build runData --console=plain` after any restoration that touches datagen, recipe codecs, or excluded source sets.
